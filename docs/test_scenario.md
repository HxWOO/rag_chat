# RAG Q&A 챗봇 시스템 테스트 시나리오

---

## 1. 문서 개요

### 1.1. 테스트 목적
본 문서는 두산 밥캣 프리미엄 지게차 매뉴얼에 대한 사용자의 질의에 답변하기 위해 설계된 RAG(Retrieval-Augmented Generation) Q&A 챗봇 시스템의 기능과 성능을 검증하는 것을 목적으로 합니다. 주요 평가 항목은 제공되는 답변의 정확성, 관련성 및 일관성입니다.

### 1.2. 테스트 범위
- **범위 내 (In-Scope):**
    - 지게차의 작동, 안전, 유지보수 등 다양한 유형의 질문에 대한 응답 품질 평가.
    - RAG 파이프라인의 종단 간(End-to-End) 검증 (질문 입력 → 관련 문서 검색 → 답변 생성 → 답변 출력).
    - 색인된 매뉴얼 내의 정보에 기반한 답변의 정확성 검증.

- **범위 외 (Out-of-Scope):**
    - 프론트엔드 UI/UX 테스트 (버튼 동작, 레이아웃 등).
    - 백엔드 인프라의 부하 테스트 또는 스트레스 테스트.
    - 인증 및 사용자 관리 기능 테스트.

### 1.3. 테스트 환경
- **애플리케이션:** Streamlit 기반 프론트엔드 (`streamlit_app.py`)
- **백엔드:** RAG 파이프라인 API (Mock 또는 실제 환경)
- **언어 모델:** Amazon Bedrock에서 제공하는 Foundation Model
- **벡터 데이터베이스:** Amazon OpenSearch Serverless

## 2. 테스트 데이터
- **소스 문서:** `두산 밥캣 프리미엄 지게차 매뉴얼.pdf`
- **개요:** 이 문서는 테스트 대상 RAG 시스템에 색인되며, 모든 테스트 시나리오는 이 지게차 매뉴얼의 내용을 기반으로 답변이 생성됨을 전제로 합니다.
